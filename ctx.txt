# GLOBAL LEADER-FOLLOWER TELEOPERATION SYSTEM SPECIFICATION

Build a complete global teleoperation system for a 6DOF robotic arm.

## Objective

Implement a real-time leader-follower teleoperation platform where:

* A leader arm is controlled by a human operator.
* A follower arm mirrors the leader arm.
* Leader and follower may be located anywhere in the world.
* Leader and follower are NOT on the same local network.
* The system must operate through the public internet.

The platform must support future expansion to humanoid robots, mobile manipulators, imitation learning datasets, and vision-language-action systems.

---

# Hardware Configuration

## Leader Station

Components:

* 6DOF Leader Arm
* Control Computer
* Display
* Keyboard and Mouse

Responsibilities:

* Read leader arm joint states
* Publish teleoperation commands
* Display live video feeds
* Display robot state and diagnostics

---

## Follower Station

Components:

* 6DOF Follower Arm
* Parallel Gripper
* Jetson Orin Nano
* Global Camera
* Wrist/Gripper Camera

Responsibilities:

* Receive teleoperation commands
* Execute robot motion
* Publish robot state
* Stream camera feeds
* Record demonstrations

---

# Network Architecture

Leader and follower must communicate over the internet.

Do not assume:

* Shared LAN
* Static IP addresses
* Open ports
* Direct peer-to-peer access

Support:

* NAT traversal
* Dynamic networks
* Connection recovery
* Session restoration

# Safety System

Implement:

## Joint Limits

* Soft limits
* Hard limits

## Workspace Limits

Configurable operating volume.

## Motion Validation

Reject:

* Invalid commands
* Unsafe poses
* Out-of-range targets

## Emergency Stop

Support:

* Hardware stop
* Software stop

---

# Failure Recovery

Detect:

* Camera failures
* Robot communication failures
* Network interruptions
* Teleoperation disconnects

On communication loss:

1. Stop robot motion.
2. Hold current pose.
3. Disable command execution.
4. Wait for reconnection.
5. Resume safely after validation.
